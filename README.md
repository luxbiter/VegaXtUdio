# Vega X U+ Boot Animation Converter

This tool converts a normal Android boot animation folder made of PNG frames into the Pantech Vega X U+ / EF14L Android 2.3.4 boot animation format.

The stock U+ `bootanimation` binary does not reliably play standard PNG boot animations. It expects `.lz` frames containing `ATC_RGB_AMD` compressed texture data, then packed into an uncompressed `bootanimation.zip`.

## Aseprite Requirements

Set these before exporting. These are not suggestions; the converter and the phone expect them.

- Canvas size: `480x800`
- Pixel aspect: square pixels
- Background: opaque black or fully drawn opaque pixels
- Color mode: `RGB Color`
- Frame size: every frame must stay exactly `480x800`
- Export format: PNG sequence
- Interlacing: off
- Scaling: `100%`
- Filename padding: use stable numeric order, for example `001.png`, `002.png`, `003.png`
- Animation rate: match `desc.txt`; the current example uses `20` fps

Recommended Aseprite export layout:

```text
BootAnim/
  desc.txt
  part0/
    001.png
    002.png
    003.png
  part1/
    000.png
```

Use `part0` for the intro animation and `part1` for the loop or final hold section. The folder names must match `desc.txt`.

## desc.txt

For Vega X U+ use:

```text
480 800 20
p 1 0 part0
p 0 2 part1
```

The first line is `width height fps`.

The `p 1 0 part0` line plays `part0` once. The `p 0 2 part1` line loops `part1` with a 2-frame pause, matching the stock U+ animation.

## Convert

Run from this folder:

```sh
node scripts/convert_bootanim_lz.js --input BootAnim --output bootanimation.zip
```

This creates:

```text
BootAnim_lz/
  desc.txt
  part0/*.lz
  part1/*.lz

bootanimation.zip
```

The zip is written in `STORE` mode, not deflated. This matters on Android 2.3 boot animation loaders.

## Options

```text
--input <dir>     Source boot animation folder. Must contain desc.txt, part0, part1.
--workdir <dir>   Folder for generated .lz frames. Default: <input>_lz
--output <zip>    Output bootanimation.zip path. Default: bootanimation.zip
--copy-to <zip>   Also copy the output zip to another path.
--help            Show CLI help.
```

Example:

```sh
node scripts/convert_bootanim_lz.js --input MyAnim --workdir build/MyAnim_lz --output build/bootanimation.zip
```

## Install On Device

Common test path:

```sh
adb push bootanimation.zip /data/local/bootanimation.zip
adb shell chmod 644 /data/local/bootanimation.zip
adb shell bootanimation
```

System path:

```sh
adb remount
adb push bootanimation.zip /system/media/bootanimation.zip
adb shell chmod 644 /system/media/bootanimation.zip
```

## Notes

- This is for the Vega X U+ / EF14L Gingerbread boot animation binary.
- The converter uses a practical ATC RGB encoder tuned for boot logos and pixel animation. It is lossy by design.
- Very thin one-pixel text can soften because ATC compresses each `4x4` block. Larger lettering, high contrast shapes, and simple gradients survive best.
- If `bootanimation` segfaults, make sure the zip contains `.lz` frames, not `.png` frames, and that it was generated by this tool.
